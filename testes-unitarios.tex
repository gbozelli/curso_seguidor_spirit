\section{Conceitos BГЎsicos}

\subsection{DefiniГ§ГЈo}

Um \textbf{teste unitГЎrio} Г© um teste pequeno e focado que verifica se uma funГ§ГЈo ou mГіdulo

\section{Por Que Testar?}

\subsection{Problemas sem testes}

Sem testes unitГЎrios, vocГӘ enfrenta vГЎrios problemas:

\begin{itemize}
    \item \textbf{Debugging demorado:} Descobrir bugs apenas apГіs carregar no hardware
    \item \textbf{RefatoraГ§ГЈo arriscada:} Medo de quebrar algo ao modificar cГіdigo
    \item \textbf{Ciclo lento:} Recompile $\rightarrow$ Upload $\rightarrow$ Teste $\rightarrow$ Repita
    \item \textbf{DocumentaГ§ГЈo ruim:} Incerteza sobre como as funГ§Гөes devem funcionar
\end{itemize}

\subsection{Vantagens com testes}

Com testes unitГЎrios implementados:

\begin{itemize}
    \item \textbf{Feedback instantГўneo:} Testes rodam em segundos no computador
    \item \textbf{ConfianГ§a para refatorar:} Mude cГіdigo sabendo que testes validam
    \item \textbf{DocumentaГ§ГЈo viva:} Testes mostram exatamente como usar cada funГ§ГЈo
    \item \textbf{Bugs detectados cedo:} Antes de enviar ao hardware
\end{itemize}

\textbf{Tarefa} & \textbf{Sem Testes} & \textbf{Com Testes} \\

\section{Boas PrГЎticas}

\subsection{1. Um conceito por teste}

Cada teste deve verificar apenas uma coisa. Se falhar, vocГӘ sabe exatamente o quГӘ quebrou.

\begin{lstlisting}[caption=Errado vs Correto]

    controlarMotor(100, 100);
    TEST_ASSERT_EQUAL_INT(100, getVelocidade());
    pararMotores();
    TEST_ASSERT_EQUAL_INT(0, getVelocidade());
}

// CORRETO: Um conceito por teste
void test_motor_controla_velocidade(void) {
    controlarMotor(100, 100);
    TEST_ASSERT_EQUAL_INT(100, getVelocidade());
}

void test_motor_para_com_sucesso(void) {
    pararMotores();
    TEST_ASSERT_EQUAL_INT(0, getVelocidade());
}
\end{lstlisting}

\subsection{2. Nomes descritivos}

Nomes de testes devem descrever exatamente o que testam.

\begin{lstlisting}[caption=Nomes de testes]

\end{lstlisting}

\subsection{3. Teste casos extremos}

Sempre teste limites: mГӯnimo, mГЎximo, negativo, overflow.
